# Watson

本地运行的日程日历 Web 应用：在浏览器中管理月度日程，数据保存在本机，无需后端。

## 项目实现了什么

- **月视图日历**：按周展示当前月份，可切换上/下一月；当日格子高亮。
- **日程管理**：支持添加、编辑、删除；优先级分为低 / 中 / 高，在界面中用颜色区分。
- **三种安排方式**
  - **未定日期**：无日期、无具体时段的任务放在左侧 **侧边栏（收件箱）**。
  - **仅有日期**：选定日期、勾选「无具体时段」时，任务只出现在对应日期格中，不绑定时刻。
  - **日期 + 时段**：填写开始/结束时间后，在当日列表中按时间排序展示。
- **日历格展示**：每个有任务的日期以 **条状色块** 显示标题（文字固定字号、过长自动截断）；**胶囊形** 表示已设时段，**圆角矩形** 表示尚未设具体时段。
- **拖拽**
  - 从侧边栏拖到某日：任务落到该日，**默认清空时段**（变为「仅有日期」）。
  - 从日历条拖回侧边栏：任务变为 **未定日期且无时段**。
- **数据持久化**：使用 `localStorage`（键名 `watson_schedules`），刷新或关闭页面后数据仍保留。

## 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | React 18 |
| 语言 | TypeScript |
| 构建 | Vite 5 |
| 样式 | Tailwind CSS 3、PostCSS |
| 状态 | Zustand |
| 日期 | dayjs（含中文区域、`isoWeek` 周布局） |

项目为纯前端静态资源，可离线打开构建产物（`dist`），也可通过 `vite preview` 本地预览。

## 如何使用

### 环境要求

- [Node.js](https://nodejs.org/)（建议 LTS，含 `npm`）

### 安装依赖

```bash
npm install
```

### 开发调试

```bash
npm run dev
```

在终端提示的本地地址（一般为 `http://localhost:5173`）用浏览器打开即可。

### 生产构建

```bash
npm run build
```

输出目录为 `dist/`，可将其中文件部署到任意静态站点托管，或直接用浏览器打开 `dist/index.html`（需注意部分环境对 `file://` 的限制；推荐用本地静态服务器或 `preview`）。

### 预览构建结果

```bash
npm run preview
```

用于在本地以 HTTP 方式查看 `npm run build` 后的效果。

## 简要操作说明

- 点击 **日期格**：查看、编辑或删除当日的日程。
- 右下角 **「+」**：添加日程；可在表单中勾选「未定日期」或「无具体时段」。
- **侧边栏**：管理未定日期任务；支持拖到日历、从日历拖回。

更完整的产品与交互说明可参考仓库中的 `watson_design.md`。
